1. Introduction

StatusEdited
RationaleUpdated capabilities list; added role-based context

Purpose

The Student Transportation Incident Tracker is your comprehensive tool for managing all incidents that occur during student transportation. Built on the Tyler Forge design system and integrated into Student Transportation (powered by Traversa), the system helps safety coordinators, supervisors, and administrators track behavioral incidents, manage workflows, communicate with drivers, monitor fleet vehicles, and generate reports.

What Can You Do?

  • Track Incidents: Record and monitor student and driver incidents across 10 categories and 35+ specific types
  • Personal Triage: Dashboard shows your assigned incidents, priority queue, and unanswered driver messages
  • Manage Students: View student profiles and their incident histories
  • Monitor Fleet: Track vehicle status, maintenance, GPS/AVL, and incidents
  • Oversee Drivers: Manage driver information, certifications, safety ratings, and performance scores
  • Manage Workflows: Create, configure, and execute multi-step workflow templates
  • Generate Reports: Run 8 quick reports with preview and CSV export
  • Communicate: Incident-linked messaging hub between coordinators and drivers
  • Get Help: In-app help center with Getting Started, User Guide, and FAQ

16. Global UI Components

Application uses Radix UI primitives (shadcn/ui) with Tyler Forge styling. Key components: Button, Card, Dialog (with forwardRef), Badge, Select, Input, Textarea, Checkbox, Avatar, Tabs, Accordion, Command, Popover, DropdownMenu, Tooltip, ScrollArea. Toasts via Sonner 2.0.3.

17. Search, Filters & Table Behavior

All search is client-side, case-insensitive, substring matching. Filters combine with AND logic. Tables support single-column sorting via header click (ascending → descending → default). No pagination in prototype.

19. Workflow System Reference

Auto-assignment via assignWorkflowToIncident(type, severity). Falls back to WF-DEFAULT. Step statuses: Not Started, In Progress, Completed, Pending Approval, Approved, Rejected. Trigger types: manual, auto-complete, time-delay, status-change, approval-granted, conditional.

20. Incident Type Taxonomy

35+ incident types across 10 categories. Student types: Behavioral (6), Safety Violation (5), Aggression/Violence (4), Driver Defiance (3), Property Damage (1), Prohibited Items (4), Privacy Violation (1). Driver types: Operational (5), Safety (4), Vehicle Incident (9). Each type has a defaultSeverity that auto-populates the severity field.

21. Notifications & Integrations

In-app triggers: new incident, driver response, unanswered comms (>24h), workflow step complete, approval required. Email notifications configurable per workflow step. TODO Webhook integration not implemented.

29. Tips & Best Practices

  • Start each day on the Dashboard reviewing My Incidents
  • Address critical-priority items first
  • Respond to driver communications within 24 hours
  • Complete workflow steps in order with comments for audit trail
  • Use filters and search to manage large incident lists
  • Protect student information per FERPA guidelines
