Design System Documentation

Complete design system guide with Tyler Forge 3.x tokens and custom branding

Overview

The Student Transportation Incident Tracker implements the Tyler Forge 3.x Design System with custom district branding. This design system ensures consistency, accessibility, and maintainability across the entire application.

Design Principles

  • Consistency: Uniform visual language throughout
  • Accessibility: WCAG AA compliant
  • Scalability: Design tokens allow easy updates
  • Brand Alignment: District colors integrated seamlessly
  • Tyler Forge Compliance: Follows Workforce application standards

Brand Identity

The application features custom district branding extracted from the transportation department logo, featuring a harmonious palette of deep blues and natural olives.

Spacing Guidelines

  • Page padding: xlarge (32px)
  • Section margins: large (24px)
  • Card padding: medium to large (16-24px)
  • Element gaps: small to medium (8-16px)

Components

Button Component

Button variants and their usage:

  • Primary (default): Main actions, important operations
  • Secondary: Alternative actions
  • Outline: Tertiary actions
  • Destructive: Delete or dangerous actions
  • Ghost: Minimal emphasis

Badge Component

Badge variants for status indication:

  • Default: Standard information
  • Secondary: Less important status
  • Destructive: Critical or error states
  • Custom Colors: Success (green), Warning (orange), Info (blue)

Card Component

Cards are the primary content container:

  • Rounded corners (0.75rem)
  • Shadow (elevation-1)
  • Padding (1.5rem)
  • White background

Avatar Component

Display user or item images:

  • Sizes: Small (32px), Medium (40px), Large (48px)
  • Fallback: Initials on colored background
  • Shape: Circular for people/items

Accessibility

Color Contrast

All text meets WCAG AA standards (4.5:1 for normal text, 3:1 for large text)

Keyboard Navigation

  • All interactive elements are keyboard accessible
  • Visible focus indicators on all focusable elements
  • Logical tab order throughout the application

ARIA Labels

  • Proper labeling for screen readers
  • Descriptive button labels
  • Form labels associated with inputs
  • Icon-only buttons have aria-label attributes

Screen Reader Support

  • Hidden text for additional context
  • Decorative elements marked as aria-hidden
  • Live regions for dynamic content
