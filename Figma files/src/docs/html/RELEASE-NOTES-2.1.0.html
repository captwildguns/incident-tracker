✨ What's New in v2.1.0

This release includes significant enhancements to the Reports page, particularly the Weekly Trends Analysis report, which has been transformed from a simple table into a comprehensive visual dashboard with charts and automated insights.

📊 Reports Page Enhancements

Enhanced Weekly Trends Analysis - Visual Dashboard

The Weekly Trends Analysis report has been completely redesigned to provide actionable insights through visualizations instead of raw data tables.

Summary Cards (4 KPI Tiles)

Card Metric Features
This Week Current week incident count Week-over-week % change with directional arrow (up/down)
Last Week Previous week count Week start date reference
High Severity Trend High severity incidents Week-over-week % change with visual indicators
4-Week Average Rolling average Incidents per week calculation

Charts & Visualizations

  • Weekly Incident Trend (Line Chart) - Displays total incidents and high severity incidents over time with color-coded lines
  • This Week by Type (Progress Bars) - Shows top 5 incident types with counts and percentages
  • Severity Distribution by Week (Stacked Bar Chart) - Visualizes High/Medium/Low severity breakdown per week

Key Insights Section

Automated insights that dynamically appear based on data patterns:

  • Significant Increase Alert - Appears when week-over-week change > 10%, displayed in red with AlertTriangle icon
  • Positive Trend - Appears when week-over-week change < -10%, displayed in green with TrendingUp icon
  • Most Common Type - Highlights incident type with > 2 occurrences this week, shows percentage breakdown

Technical Implementation

  • Built with Recharts library for responsive, interactive charts
  • All styling uses Tyler Forge design system CSS variables (var(--forge-*))
  • Unique data keys prevent React duplicate key warnings
  • Responsive containers adapt to different screen sizes

Changed Report Column Structure Updates

Two reports have been updated with improved column structures for better readability:

Report Previous Structure New Structure
Driver Report Summary Incident ID → Date → Student → Type → Vehicle → Driver → Severity → Status Incident ID → Date → Driver → Type → Vehicle → Severity → Status
(Student column removed, Driver after Date)
Vehicle Incident Report Incident ID → Date → Student → Type → Vehicle → Driver → Severity → Status Incident ID → Date → Driver → Type → Vehicle → Severity → Status
(Student column removed, Driver after Date)

Rationale: These reports focus on driver and vehicle performance metrics, so prioritizing driver information provides more relevant context than student details.

📱 Communications Page

Added Unread Filter Option

The Communications page now includes an "Unread" filter option, allowing safety coordinators to quickly identify and respond to messages that haven't been read yet.

  • Filter by Unread status
  • Visual indicator for unread messages
  • Improves response time and workflow efficiency

📋 Incidents Page

Enhanced Clickable Table Rows

Incident table rows are now fully clickable, providing improved navigation to incident details.

  • Click anywhere on a row to view incident details
  • Hover state provides visual feedback
  • Consistent with other pages (Vehicles, Students, Drivers)

🚌 Vehicles Page

Added Full Pagination Support

The Vehicles page now includes complete pagination for all 15 vehicles in the fleet.

  • 10 vehicles per page
  • Previous/Next navigation
  • Clickable table rows for vehicle details

🐛 Bug Fixes

Fixed React Key Warnings in Charts

Resolved duplicate key warnings in Recharts components by adding unique identifiers to all chart data points:

  • Line chart data: id: `week-${index}`
  • Bar chart data: id: `severity-week-${index}`
  • Eliminates console warnings
  • Improves React rendering performance

Fixed Figma Inspector Props Warnings

Applied filterFigmaProps utility function to remove Figma-specific props from production components, eliminating React warnings about unknown DOM properties.

🎨 Design System Adherence

All new features and enhancements strictly adhere to the Tyler Forge 3.x design system:

  • CSS Variables: All colors, spacing, borders, radius, and typography use var(--forge-*) variables
  • Typography: Roboto font family exclusively (var(--forge-font-family))
  • Consistent Styling: Ensures cohesive visual experience across the application
  • Customizable: Design can be updated globally by modifying CSS variables
