Technical Documentation

Developer reference for the Student Transportation Incident Tracker

Architecture Overview

The Student Transportation Incident Tracker is a single-page application (SPA) built with React and TypeScript, implementing a component-based architecture.

┌─────────────────────────────────────────┐
│         Presentation Layer              │
│  (React Components + Tyler Forge UI)   │
├─────────────────────────────────────────┤
│         State Management Layer          │
│     (React Hooks - useState)            │
├─────────────────────────────────────────┤
│         Business Logic Layer            │
│  (Filtering, Sorting, Validation)       │
├─────────────────────────────────────────┤
│         Data Layer (Mock Data)          │
│   (Future: API Integration Layer)       │
└─────────────────────────────────────────┘

Design Patterns

  • Component Composition: Small, reusable components composed into larger page components
  • Container/Presentational: Pages act as containers managing state, UI components handle presentation
  • Prop Drilling: Navigation callback passed through component tree
  • Render Props: Dialog components use render props pattern for trigger elements

Component Documentation

App Component (/App.tsx)

Purpose: Root component managing navigation and page routing

export default function App() {
  const [currentPage, setCurrentPage] = useState('dashboard');

  const handleNavigate = (page: string) => {
    setCurrentPage(page);
  };

  return (
    <div className="min-h-screen bg-background">
      {/* Omnibar */}
      <nav>...</nav>
      
      {/* Page Content */}
      {currentPage === 'dashboard' && <DashboardPage onNavigate={handleNavigate} />}
      {/* ... other pages */}
    </div>
  );
}

Page Components

All page components follow a consistent pattern:

  • Accept onNavigate prop for navigation
  • Manage local state with useState hooks
  • Implement filtering and sorting logic
  • Use Tyler Forge UI components

Sortable Table Implementation

const [sortColumn, setSortColumn] = useState('date');
const [sortDirection, setSortDirection] = useState('desc');

const handleSort = (column) => {
  if (sortColumn === column) {
    setSortDirection(sortDirection === 'asc' ? 'desc' : 'asc');
  } else {
    setSortColumn(column);
    setSortDirection('asc');
  }
};

const SortIcon = ({ column }) => {
  if (sortColumn !== column) {
    return <ChevronsUpDown className="h-4 w-4 ml-1 text-muted-foreground" />;
  }
  return sortDirection === 'asc' 
    ? <ChevronUp className="h-4 w-4 ml-1" /> 
    : <ChevronDown className="h-4 w-4 ml-1" />;
};

State Management

The application uses React's built-in useState hook for state management. Each page component manages its own local state.

const [searchTerm, setSearchTerm] = useState('');
const [statusFilter, setStatusFilter] = useState('all');
const [sortColumn, setSortColumn] = useState('date');
const [sortDirection, setSortDirection] = useState('desc');

Data Models

Incident Model

interface Incident {
  id: string;                    // Unique identifier
  date: string;                  // ISO date string
  time: string;                  // HH:MM format
  studentName: string;
  studentId: string;
  driverName: string;
  vehicle: string;
  type: 'Student Behavior' | 'Safety Violation' | 'Mechanical' | 'Accident' | 'Other';
  severity: 'Critical' | 'High' | 'Medium' | 'Low';
  status: 'Open' | 'Under Review' | 'Resolved' | 'Closed';
  description: string;
  actionsTaken: string;
  location: string;
}

Student Model

interface Student {
  id: string;
  name: string;
  photoUrl: string;
  grade: string;
  school: string;
  contact: string;
  route: string;
  incidents: Incident[];
  activeIncidents: boolean;
}

Vehicle Model

interface Vehicle {
  id: string;
  name: string;
  make: string;
  model: string;

  status: 'Active' | 'Maintenance' | 'Inactive';
  driver: string;
  maintenanceStatus: 'Excellent' | 'Good' | 'Needs Attention' | 'In Repair';

  mileage: number;

Development Guide

Setup Development Environment

# Clone repository
git clone <repository-url>
cd incident-tracker

# Install dependencies
npm install

# Start development server
npm run dev

# Open browser
# Navigate to http://localhost:5173

Adding a New Page

  1. Create component file: src/components/your-page/YourPage.tsx
  2. Define component with onNavigate prop
  3. Add to App.tsx navigation and routing

Code Style Guidelines

  • Use TypeScript for all new files
  • Define interfaces for props and data models
  • Use functional components with hooks
  • Follow Tyler Forge design system guidelines
  • Use CSS variables for styling

Build and Deployment

Building for Production

npm run build

This creates optimized production files in /dist/ directory.

Deployment Options

  • Vercel: vercel deploy
  • Netlify: netlify deploy --prod
  • Static Hosting: Copy dist folder to web server
